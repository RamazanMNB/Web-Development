# OOP Python Project: Animal Hierarchy

## Описание проекта

Этот проект демонстрирует использование **объектно-ориентированного программирования (OOP)** в Python.
Создана иерархия классов, моделирующая животных: базовый класс `Animal` и дочерние классы `Dog` и `Cat`.

Проект показывает:

* Наследование
* Переопределение методов (override)
* Полиморфизм
* Работа с методами и атрибутами объектов

---

## Структура проекта

```
project/
│
├── models.py      # Файл с классами Animal, Dog и Cat
├── main.py        # Основной скрипт для демонстрации работы классов
└── README.md      # Этот файл
```

---

## Классы

### 1. Animal (базовый класс)

* Атрибуты: `name`, `age`, `color`
* Методы:

  * `eat()` — демонстрирует, что животное ест
  * `speak()` — демонстрирует звук животного
  * `__str__()` — строковое представление объекта

### 2. Dog (дочерний класс)

* Наследует `Animal`
* Дополнительный атрибут: `stray` (породистый или бродячий)
* Переопределяет методы: `eat()` и `speak()`

### 3. Cat (дочерний класс)

* Наследует `Animal`
* Дополнительный атрибут: `breed` (порода)
* Переопределяет методы: `eat()` и `speak()`

---

## Как использовать

1. Убедитесь, что оба файла (`models.py` и `main.py`) находятся в одной папке.
2. Запустите `main.py`:

```bash
python main.py
```

3. Вы увидите вывод всех объектов, их атрибутов и работу методов.

---

## Пример вывода

```
Name: JustAnimal, Age: 10, Color: Red
JustAnimal is eating

Rex is eating a bone
Rex barks

Name: Murka, Age: 4, Color: White, Breed: Persian
Murka is eating fish

```

---

## Особенности проекта

* Демонстрируется **наследование**: Dog и Cat наследуют свойства и методы Animal
* Демонстрируется **переопределение методов**: каждый класс имеет свой `speak()` и `eat()`
* Показывается **полиморфизм**: один и тот же метод `speak()` работает по-разному для разных объектов
